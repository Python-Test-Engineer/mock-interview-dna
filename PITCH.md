# Data Intelligence Researcher

**Your dataset. Your questions. Claude does the rest.**

---

Most companies sit on data they can't act on — not because insight is impossible, but because the gap between "I have a question" and "I have an answer" is too wide. Hiring data scientists is slow and expensive. BI tools require expertise. Dashboards answer last month's questions.

This changes that.

---

## What it does

- **You write a rough idea** — bullet points, a sentence, a hunch. No formal spec required.
- **Claude plans the analysis** — profiles your data, asks the right questions, writes a research plan.
- **Claude architects the solution** — scripts, data contracts, output structure. Review it before a line of code runs.
- **Claude executes everything** — writes Python, runs it, fixes errors, validates outputs.
- **You get results** — plots, cleaned datasets, statistical tables, and a self-contained HTML report. Every time.

One command per step. No boilerplate. No setup beyond the first `uv sync`.

---

## Why it becomes the default

- **Any analyst can use it** — if you can write a few bullet points, you can run a full analysis.
- **Reproducible by design** — every dirty row is logged, every output is validated, every run is auditable.
- **Extensible** — dashboards, domain knowledge agents, code review, and insights are all pluggable skills.
- **No lock-in** — it's Python all the way down. You own the code it writes.
- **Scales with the team** — a solo analyst gets PhD-level output; a team gets a shared, consistent workflow.

---

## The compounding effect

The first run saves hours. By the tenth, it's changed how the team thinks about data — from "we'll look into that eventually" to "let's find out right now." That shift is the real product.
